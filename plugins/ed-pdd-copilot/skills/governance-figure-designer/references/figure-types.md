# Governance Figure Types

| Figure type | Use when | Core visual question |
|---|---|---|
| Macro governance framework | Explaining policy, organization and technology relationship | What levels and values guide the system? |
| Layered smart campus architecture | Presenting platform or Agent architecture | How do data, intelligence, applications and oversight connect? |
| Multi-actor collaboration diagram | Showing administrators, teachers, students, systems and experts | Who acts, decides, reviews and is accountable? |
| Human-AI feedback loop | Explaining decision support or evaluation | Where does human interpretation and override happen? |
| Indicator system tree | Presenting evaluation framework | How are dimensions and indicators organized and validated? |
| Evidence chain diagram | Explaining mixed-methods validation | How do multiple data sources support claims? |
| Data governance responsibility map | Explaining data rights, stewardship and accountability | Who owns, manages, accesses, audits and is affected by data? |
| Teacher development support mechanism | Explaining teacher digital competence governance | How do region, school,教研 group and teacher form a support loop? |
| Human-AI assessment governance loop | Explaining AI-supported formative assessment | Where do AI suggestion, teacher interpretation, student feedback and appeal connect? |

## Figure Selection Rules

- Use a **responsibility map** when the paper discusses data governance, privacy, risk or accountability.
- Use a **support mechanism loop** when the paper discusses teacher development, capacity building or organizational change.
- Use a **human-AI governance loop** when the paper discusses decision support, assessment, recommendation or early warning.
- Use an **evidence chain** when the figure must prove how mixed evidence supports different claims.
