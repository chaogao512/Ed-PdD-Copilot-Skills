# Method Reporting Templates

Use these templates when the user needs a concrete methods section, not only a design idea.

## 1. Delphi Reporting Template

| Item | Required content |
|---|---|
| Purpose | What consensus is needed and why empirical evidence alone is insufficient |
| Expert criteria | Field, years of experience, publication/practice record, institutional diversity |
| Panel size | Number invited, number completed each round, attrition |
| Rounds | Round 1 item generation/rating; Round 2 feedback and revision; Round 3 confirmation if needed |
| Anonymity | How responses are anonymized and summarized |
| Consensus threshold | Mean/median, coefficient of variation, IQR, agreement percentage or predefined rule |
| Revision logic | Which items were retained, merged, deleted or reworded and why |
| Output | Final indicator/model/framework and unresolved disagreements |

## 2. Questionnaire Reporting Template

| Item | Required content |
|---|---|
| Construct | Theory source and definition |
| Instrument source | Existing scale, adapted scale or self-developed items |
| Sample | Population, sampling method, valid responses, demographics |
| Reliability | Cronbach's alpha or other reliability index when appropriate |
| Validity | Content validity, construct validity, EFA/CFA or expert review |
| Analysis | Descriptive, group comparison, correlation, regression or model testing |
| Boundary | What the questionnaire can and cannot prove |

## 3. Interview Reporting Template

| Item | Required content |
|---|---|
| Participant selection | Role, experience, sampling strategy |
| Protocol | Main question areas and relation to RQs |
| Data collection | Time, mode, recording, transcription |
| Coding | Open/axial/theme coding or predefined framework |
| Trustworthiness | Double coding, member checking, audit trail, negative cases |
| Evidence use | Representative quotes and relation to quantitative findings |

## 4. Log/Data Trace Reporting Template

| Item | Required content |
|---|---|
| Source system | Platform or process generating data |
| Data dictionary | Variables, timestamps, actor roles, event meaning |
| Cleaning | Missing data, duplicate records, abnormal cases |
| Metrics | How each metric corresponds to a governance claim |
| Privacy | De-identification, access control, retention |
| Interpretation boundary | What logs cannot reveal without qualitative evidence |
