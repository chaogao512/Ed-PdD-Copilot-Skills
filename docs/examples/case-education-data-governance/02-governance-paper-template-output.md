# Education Data Governance Case: Paper Skeleton

## Skill Invoked

`governance-paper-template`

## Paper Type

治理框架 + 风险防控机制论文。

## Thesis Statement

高校教育数据治理能力不是数据汇聚能力，而是围绕权责、标准、质量、安全、伦理和反馈形成的组织化治理能力。

## Matrix

| Theory | Mechanism | Practice Element | Evidence |
|---|---|---|---|
| 数据治理理论 | 权责、标准、质量、生命周期 | 数据目录、授权流程、质量规则 | 制度文本、流程记录、质量指标 |
| 协同治理理论 | 多部门共治与冲突协调 | 数据治理委员会、跨部门议事机制 | 访谈、会议记录、协同案例 |
| 风险治理理论 | 风险识别、评估、缓释、监测 | 风险台账、伦理审查、审计追踪 | 风险案例、审计记录 |

## Section Outline

1. 引言：从数据平台建设到教育数据治理能力。
2. 文献与政策背景：教育数据治理、智慧校园和 AI 风险治理。
3. 理论框架：能力维度与风险防控逻辑。
4. 研究设计：文本分析、访谈、专家咨询和流程审计。
5. 能力框架构建：战略、权责、标准、质量、安全、伦理、反馈。
6. 风险防控机制：数据最小化、用途限定、人工复核、申诉纠错。
7. 案例验证与修正。

## Quality Self-Check

论文贡献落在制度和机制，不把平台建设当作治理本身。
